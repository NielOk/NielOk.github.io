A Computational View on the Asymmetry of Time

Why does time move forward and not backward? I have a perspective that doesn't rely on thermodynamics or psychology. Instead, I'd like to argue from a computational perspective. Specifically, when we think of the universe as a computational system that applies physical laws to its current state to produce the next, we can see that the direction of time emerges from the computational constraints of this system, as long as we put some constraints on the computer.

Defining the Universe as a Stateless Computer

Let's define what this computer looks like. At any given moment, it has access to the complete state of the universe at that moment, which we can denote as \(S_t\). This state includes all particles, fields, and their interactions. The computer applies the physical laws, which we can represent as a function \(f\), to this state to compute the next state of the universe:

\[ S_{t+1} = f(S_t) \]

The computer is stateless in that it doesn't retain any information about previous states \(S_{t-1}\) or the history of the universe. All it does is take the current state and apply the transition function to compute the next state. The computer is constrained to one instance of the universe, meaning it can't access parallel branches, even if the many-worlds interpretation is true.

Notice that, if the transition function \(f\) were bijective (i.e., one-to-one and onto), the computer could also compute the previous state by applying the inverse function:

\[ S_{t-1} = f^{-1}(S_t) \]

This would imply that the computer could simulate both forward and backward in time with equal ease, as each state would have a unique predecessor and successor.

An Irreversible System

At this point, you'll notice that all I've done is set up a framework for computing the universe, but I haven't yet explained why that computation moves in a specific direction. Don't worry, we're here.

I'd like to argue that \(f\) is not bijective. In fact, it is a many-to-one function, meaning that multiple distinct states can evolve into the same next state. Formally, this means

\[ \exists S_{t-1}^{(1)} \neq S_{t-1}^{(2)} : f(S_{t-1}^{(1)}) = f(S_{t-1}^{(2)}) = S_t \]

Why? There is one fundamental irreversible system given our current definition of the computer: quantum measurement. When a quantum system is measured, it collapses from a superposition of states into a single outcome, losing information about the prior superposition. This process is inherently many-to-one, as multiple distinct superpositions can yield the same measurement result. If the computer had access to every instance of the universe under the many-worlds interpretation, this would not be a problem, as it could access the full superposition. However, since it is constrained to one instance, the information loss from measurement is irreversible from the computer's perspective.

This renders the inverse computation ill-defined, as the computer has no mechanism to distinguish or automatically select the correct precursor among multiple plausible candidates to the current state. Thus, the computer can only reliably compute forward in time.

Conclusion

Although this isn't a proof of the asymmetry of time, I think it's an interesting computational perspective on the problem. I hope to build on this perspective as I learn more. This was just my gut reaction when I first became aware that time only moves forward.
